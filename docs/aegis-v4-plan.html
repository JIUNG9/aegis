1 Layer 1: LLM Wiki (Karpathy Pattern)

Key Concept: LLM Wiki vs Traditional RAG

Traditional RAG chunks docs and searches chunks every query — no accumulation, no staleness detection. LLM Wiki (Andrej Karpathy's pattern) has the AI read sources ONCE, build structured wiki pages, detect contradictions, and compound knowledge over time.

❌ Traditional RAG

  • Chunks docs → embeds → searches at query time
  • No staleness detection — serves 2-year-old Confluence pages as truth
  • No knowledge accumulation — rediscovers every query
  • 90% accuracy (lost-in-the-middle)
  • Cost: $30-100/month (cloud embeddings + vector DB)

✅ LLM Wiki

  • AI reads source → merges into existing wiki pages
  • Contradiction flags — new source conflicts → flagged immediately
  • Knowledge compounds — each doc updates, not re-chunks
  • Higher accuracy — pre-synthesized, cross-referenced
  • Cost: $0.50-2/month (local Haiku synthesis)

Time-Based Pattern Detection

The Pattern Analyzer aggregates SigNoz data over time and detects recurring patterns:

  • Peak hours: "Traffic peaks at 9-11 AM and 2-4 PM KST"
  • Weekly cycles: "80% of payment-service errors occur on Monday mornings"
  • Seasonal trends: "Auth latency increases during batch jobs at 02:00"
  • Scaling opportunities: "Traffic drops 70% between 22:00-06:00 — scale down"
  • Deployment correlation: "3 of last 5 incidents followed a Friday deploy"

4 Layer 4: Production Guardrails

⚠️ Core Principle: Observe First, Act Later

Start in observation mode. Build trust with the team. Only graduate to automation after weeks of accurate recommendations.

The 4-Stage Automation Ladder

Stage 1

OBSERVE

AI watches, logs findings. No recommendations shown to users.

Stage 2

RECOMMEND

AI shows recommendations but cannot act. Humans decide.

Stage 3

LOW-AUTO

AI auto-acts on low-risk. Slack approval for medium/high.

Stage 4

FULL-AUTO

AI auto-acts low+medium. Manual only for high-risk.

7 Key Concepts to Understand

🧠 Karpathy's LLM Wiki Pattern

Instead of RAG's "embed chunks → search at query time", the AI reads sources ONCE, builds structured wiki pages, detects contradictions, and compounds knowledge. Think of it as the AI maintaining your team's Confluence — but it actually stays current.

Original gist by Andrej Karpathy | llm-wiki on GitHub

🛡️ Autonomy with Guardrails (Conf42 SRE 2026)

The agent generates a plan → passes through guardrails/policy → system validates → agent interacts through secure APIs only. Free-form AI outputs converted to controlled, verifiable operations.

Conf42 SRE 2026 Talk

📊 Observation Mode → Automation Ladder

Never go straight to automation. Start by observing (the AI watches and logs). Then recommend (humans see suggestions). Then low-risk auto (AI acts on safe things). Finally full-auto with approval gates. This builds team trust progressively.

🏗️ Agent Control Plane (Galileo, open source)

Centralized governance for AI agents. AWS, CrewAI, and Glean are among the first partners. This is what Aegis's Control Tower implements.

🔄 Contradiction Detection & Staleness Lint

The wiki periodically runs "lint checks" — like spell-check for knowledge. Finds: orphaned pages with no links, stale content beyond freshness threshold, contradictions between sources, missing cross-references. This is why LLM Wiki > Confluence.
